## What is a point cloud

A set of 3D points (X, Y, Z) — optionally with color (R, G, B) or intensity — representing a surface or scene. Generated by:
- **LiDAR:** rotating laser, returns range → spherical to Cartesian
- **Depth camera (RealSense D435):** depth image + intrinsics → backproject each pixel

```python
# Depth image → point cloud (numpy)
rows, cols = depth.shape
u = np.arange(cols)
v = np.arange(rows)
uu, vv = np.meshgrid(u, v)

X = (uu - cx) * depth / fx
Y = (vv - cy) * depth / fy
Z = depth

points = np.stack([X, Y, Z], axis=-1).reshape(-1, 3)
# Filter out invalid depth (Z == 0 or Z > max_range)
points = points[points[:, 2] > 0]
```

In ROS2: `sensor_msgs/msg/PointCloud2` — packed array of XYZ (+ optional RGB, intensity).

## Voxel grid downsampling

Raw point clouds have millions of points — too slow for real-time processing. Voxel grid divides space into cubes (voxels) and keeps one representative point per voxel.

```python
import open3d as o3d

pcd = o3d.io.read_point_cloud("scan.ply")
downsampled = pcd.voxel_down_sample(voxel_size=0.05)  # 5cm voxels
```

**Why:** reduces compute while preserving shape. RTAB-Map uses `Grid/DepthDecimation` for this. Nav2 costmaps use voxel layers.

## ICP (Iterative Closest Point)

Aligns two point clouds by minimizing the distance between corresponding points. Core of LiDAR SLAM front-end.

**Algorithm:**
```
1. For each point in source, find nearest neighbor in target
2. Compute transformation (R, t) that minimizes Σ||p_i - (R*q_i + t)||²
3. Apply transformation to source
4. Repeat until convergence (change < threshold)
```

**Output:** rigid transformation (R, t) that aligns source to target.

**When ICP fails:**
- Poor initial alignment (gets stuck in local minimum) → use better initialization (NDT, feature matching)
- Too few overlapping points → robot moved too far between scans
- Planar/symmetric environments → degeneracy (no unique solution)
- Noise too high → outlier rejection (point-to-plane ICP is more robust)

**Variants:**
- Point-to-point ICP: minimize point distances (sensitive to noise)
- Point-to-plane ICP: minimize distance to surface normals (more robust, faster convergence)

## Normal estimation

Surface normals enable point-to-plane ICP and feature extraction.

```python
pcd.estimate_normals(
    search_param=o3d.geometry.KDTreeSearchParamHybrid(radius=0.1, max_nn=30))
```

Computed via PCA on local neighborhood — eigenvector with smallest eigenvalue = normal direction.

## KD-tree for nearest neighbor search

ICP needs fast nearest neighbor lookup. KD-tree partitions space recursively for O(log N) average query time.

```python
# ROS2 PCL
pcl::KdTreeFLANN<pcl::PointXYZ> kdtree;
kdtree.setInputCloud(target_cloud);
std::vector<int> indices;
std::vector<float> distances;
kdtree.nearestKSearch(query_point, 1, indices, distances);
```

## RTAB-Map point cloud pipeline

```
RealSense D435 depth image
    ↓ backproject with intrinsics
Raw point cloud (PointCloud2)
    ↓ voxel downsample (Grid/DepthDecimation)
Downsampled cloud
    ↓ feature extraction (ORB features on RGB)
    ↓ scan matching (ICP or feature-based)
Frame-to-frame odometry
    ↓ loop closure detection
    ↓ graph optimization
Global map
```

#flashcards

What is ICP and when does it fail? ? ICP (Iterative Closest Point) iteratively matches nearest-neighbor point pairs and minimizes distance to align two clouds. Fails when: poor initial alignment (local minimum), too little overlap, planar/symmetric environments (degenerate), or excessive noise. Point-to-plane ICP is more robust than point-to-point.

Why downsample a point cloud with a voxel grid? ? Raw point clouds have millions of points — too slow for real-time registration and mapping. Voxel grid keeps one representative point per cubic cell, reducing size 10-100x while preserving shape. RTAB-Map, ICP, and Nav2 voxel layers all use this.

How do you backproject a depth image pixel to a 3D point? ? X = (u - cx) * depth / fx, Y = (v - cy) * depth / fy, Z = depth. This is the inverse of the pinhole projection — requires camera intrinsics (fx, fy, cx, cy) from camera_info.
